# Absolute Travel

## Встановлення (Installation)

Встановіть усі залежності проекту однією командою:
```bash
npm run install:all
```

## Запуск (Run)

Запустіть клієнтську та серверну частини проекту одночасно:
```bash
npm run dev
```

## Карта місць та модерація (Places map & moderation)

На вкладці **«Мапа мандрівок»** відображаються місця з бекенду (`GET /api/places`).
Кожне місце має **реальну геолокацію (lat/lng)**, яку проєктовано на стилізовану
мапу України, а користувацькі місця — ще й **щонайменше 2 фото**.

**Хто може додавати місця:**
- **Будь-який користувач** — кнопка «Додати місце» на карті. Заявка проходить
  **ШІ-модерацію** (Google Gemini, з аналізом фото): адекватні місця публікуються
  автоматично, сумнівні — потрапляють на ручну перевірку, неприйнятні —
  відхиляються. Обов’язкові щонайменше 2 фото та геолокація.
- **Адміністратор** — окремий акаунт із логіном і паролем: додає місця напряму
  (без модерації), схвалює / відхиляє / видаляє заявки, а також **керує іншими
  адміністраторами** (створює та видаляє акаунти).

**Єдиний вхід:** форма входу одна для всіх. Якщо ввести логін і пароль
адміністратора — потрапляєш у **меню адміністратора** з усіма функціями вище.
Якщо це звичайний учасник — потрапляєш у звичайне меню, де **немає жодних згадок
про адмінку**.

**Адмін-акаунти:**
- **Головний адміністратор** — єдиний, фіксований акаунт. Його логін і пароль
  беруться з `.env` (`ADMIN_LOGIN` / `ADMIN_PASSWORD`) і синхронізуються під час
  кожного старту сервера. Його **не можна видалити** через панель.
- **Звичайні адміністратори** — створюються будь-яким адміном у розділі
  «Адміністратори». Мають ті самі функції модерації. Головний адмін завжди один.

Технічно: вхід повертає сесійний токен (заголовок `x-admin-token`, зберігається
у браузері). За замовчуванням: логін `admin`, пароль `admin123`.

**Налаштування (`.env`):**
```
# Увімкнути ШІ-модерацію (без ключа заявки йдуть на ручну перевірку):
GEMINI_API_KEY="ваш_ключ"          # https://aistudio.google.com/apikey
# Головний (фіксований) адмін-акаунт — змініть у продакшені на надійні значення:
ADMIN_LOGIN="admin"
ADMIN_PASSWORD="admin123"
```

> Без `GEMINI_API_KEY` застосунок працює: усі користувацькі заявки просто
> отримують статус «на перевірці» й чекають на адміністратора.
